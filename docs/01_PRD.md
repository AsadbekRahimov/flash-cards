# 01. Product Requirements Document — LexiFlow Pro

## 1. Vision

LexiFlow Pro — инструмент для языковых центров, который превращает Telegram-группу класса в полноценную обучающую среду. Учитель загружает контент через админку, запускает режимы обучения в группе, а студенты через Telegram Web App учат слова с карточками, TTS и интервальными повторениями. Цель — убрать трение между «расписанием занятия» и «домашкой»: студент получает персональное напоминание и повторяет именно те слова, которые ему трудно даются.

## 2. Проблема, которую решаем

- Учителя сейчас вручную рассылают списки слов (Google Doc / Excel / фото) — нет автоматизации и нет прогресса.
- Студенты забывают слова — нет интервальных повторений.
- Нет видимой для учителя аналитики: кто что выучил, какие слова «проваливаются» на группу.
- Отдельные приложения (Anki, Quizlet) — ещё одно приложение, которое нужно установить, зарегистрироваться, подписаться. Telegram уже есть у всех.

## 3. Целевые пользователи (Roles)

### 3.1 Администратор платформы (Admin)
- Владелец / техдир языкового центра.
- Регистрирует учителей, привязывает Telegram-группы.
- Видит общую аналитику по всем группам.

### 3.2 Учитель (Teacher)
- Ведёт группу студентов в Telegram.
- Импортирует уроки (JSON).
- Запускает тренировки и экзамены в группе.
- Смотрит прогресс по своим группам.

### 3.3 Студент (Student)
- Член Telegram-группы.
- Не регистрируется отдельно — авторизуется через Telegram.
- Открывает TWA, учит карточки, сдаёт экзамен.
- Получает от бота персональные напоминания о повторении.

## 4. Основные User Stories

### Admin
- **AS-01** Как админ, я хочу добавить нового учителя и привязать его к Telegram-группе, чтобы он мог там запускать сессии.
- **AS-02** Как админ, я хочу видеть дашборд с топ-активных групп и студентов, чтобы понимать, где продукт работает, а где нет.
- **AS-03** Как админ, я хочу видеть список «трудных слов» по группе, чтобы обсудить это с учителем.

### Teacher
- **TS-01** Как учитель, я хочу загрузить JSON с новым уроком, чтобы контент появился у студентов.
- **TS-02** Как учитель, я хочу отправить команду `/start_training` в группе, чтобы все студенты получили кнопку открытия TWA с карточками текущего урока.
- **TS-03** Как учитель, я хочу запустить экзамен на 10 минут, чтобы в конце получить таблицу лидеров прямо в групповом чате.
- **TS-04** Как учитель, я хочу видеть прогресс каждого студента по урокам, чтобы давать обратную связь.

### Student
- **SS-01** Как студент, я хочу нажать на кнопку в Telegram и открыть TWA, чтобы не ставить отдельное приложение.
- **SS-02** Как студент, я хочу слышать произношение слова, чтобы запомнить и понимать на слух.
- **SS-03** Как студент, я хочу отмечать «трудно/средне/легко», чтобы система показывала мне эти слова чаще.
- **SS-04** Как студент, я хочу получать от бота напоминание «пора повторить X слов», чтобы не забывать.
- **SS-05** Как студент, я хочу видеть таблицу лидеров после экзамена, чтобы было соревновательно.

## 5. Функциональные требования

### 5.1 Admin Panel (Filament 5)

#### FR-ADM-01 Управление пользователями
- CRUD для `User` (админы и учителя разделяются ролями через `spatie/laravel-permission`).
- Учитель может быть привязан к одной или нескольким группам (`teacher_groups`).

#### FR-ADM-02 Управление Telegram-группами
- Список групп с полями: `chat_id` (int64), `title`, `teacher_id`, `is_active`, `created_at`.
- Поле `chat_id` получаем двумя путями:
  1. Учитель добавляет бота в группу → бот ловит событие `my_chat_member` → создаёт группу в статусе `pending`.
  2. Админ вручную активирует группу.
- Только группы со статусом `is_active = true` и привязкой к учителю реагируют на команды бота.

#### FR-ADM-03 Импорт контента (JSON)
- Страница загрузки файла в Filament. Валидация схемы (см. раздел 9).
- При импорте:
  - Находим или создаём `Stage` по номеру.
  - Находим или создаём `Lesson` внутри stage.
  - Для каждого слова — upsert в `words` по `(lesson_id, word)`.
- После импорта — возврат отчёта: «добавлено X слов, обновлено Y, пропущено Z (ошибки валидации)».

#### FR-ADM-04 Дашборд аналитики
- Карточки сверху:
  - Всего студентов (distinct `students` по активным группам).
  - Всего сданных экзаменов за последние 30 дней.
  - Средняя доля правильных ответов.
- Графики:
  - Активность по дням (линейный график — сессии/день).
  - Топ-10 активных студентов (таблица).
  - Топ-20 «трудных слов» (слова, где доля `quality < 3` максимальна, с min. кол-вом попыток 10).
- Фильтр: по группе, по временному промежутку.

#### FR-ADM-05 Логи экзаменов
- Таблица: студент, группа, урок, дата, баллы, время.
- Drilldown на конкретный экзамен → видно пословную разбивку.

#### FR-ADM-06 Уроки и слова (read-only + небольшие правки)
- Просмотр `Stage → Lesson → Word` деревом.
- Возможность отредактировать `translation` или `example` у конкретного слова (без удаления).

### 5.2 Telegram Bot

#### FR-BOT-01 Group Lock
- Любое сообщение, пришедшее боту в личке — игнор (или вежливый ответ: «Я работаю только в учебных группах»).
- При сообщении в группе бот проверяет, что `chat_id ∈ groups WHERE is_active = true`. Если нет — молчит.

#### FR-BOT-02 Команда `/start_training`
- Выполняется только учителем (`from.id == groups.teacher.telegram_user_id`).
- Параметры: `/start_training <stage> <lesson>` или без параметров — тогда бот показывает inline-клавиатуру выбора урока.
- Результат: бот постит в чат сообщение «Тренировка по Уроку N началась. Нажмите кнопку ниже» с кнопкой `web_app` → открытие TWA в режиме `training`.

#### FR-BOT-03 Команда `/start_exam`
- То же авторизационное правило.
- Параметры: `/start_exam <stage> <lesson> [duration_minutes=10]`.
- Бот открывает сессию экзамена в БД со статусом `open`, постит сообщение с кнопкой `web_app`.
- По истечении `duration_minutes` (через scheduled job / `delay`) бот подсчитывает результаты и постит лидерборд.

#### FR-BOT-04 Команда `/stats` (учитель)
- Выдаёт учителю в личку (а не в группу) короткий отчёт по группе за последние 7 дней.

#### FR-BOT-05 Автоматические push-уведомления студенту
- Ежедневный cron `0 18 * * *` (локальное время группы, MVP — UTC) проверяет `word_repetitions WHERE next_review_at <= now()` по каждому студенту.
- Если у студента ≥ 5 слов готовы к повторению — отправляет личное сообщение: «Пора повторить N слов из Урока K. [Открыть]».
- Rate-limit: не чаще 1 уведомления в 24 часа на студента.

### 5.3 Telegram Web App (TWA)

#### FR-TWA-01 Авторизация
- При открытии TWA получает `window.Telegram.WebApp.initData` — строку с подписью.
- Отправляет её на backend (`POST /api/twa/auth`), backend валидирует HMAC (см. SECURITY.md).
- Backend возвращает короткоживущий JWT (15 минут) для последующих запросов.

#### FR-TWA-02 Режим «Training» (Flashcards)
- Экран карточки:
  - Слово (большим шрифтом).
  - Кнопка 🔊 (TTS).
  - Кнопка «Показать перевод» → раскрывает перевод и пример.
  - Три кнопки: 😰 «Трудно», 🙂 «Средне», 😎 «Легко».
- После оценки — следующая карточка.
- Алгоритм подбора карточек: сначала те, у которых `next_review_at <= now()`; если все повторены — новые из урока; если все выучены — «вы завершили урок».

#### FR-TWA-03 Режим «Exam»
- Таймер обратного отсчёта.
- Один вопрос — четыре варианта перевода (из них 1 правильный + 3 дистрактора из других слов этого stage).
- Баллы: базовые 10 за правильный ответ + бонус ×1.5 если слово было помечено `hard` у этого студента + штраф за превышение лимита времени на ответ (10 сек на вопрос).
- По окончании — экран «Ваш результат: X / Y, ранг #Z».

#### FR-TWA-04 TTS (Web Speech API)
- Primary: `speechSynthesis.speak(new SpeechSynthesisUtterance(word))` с `lang='en-US'` и `rate=0.9`.
- Fallback: если `speechSynthesis` недоступен — GET запрос на `/api/twa/tts?word=...`, который через edge-tts микросервис возвращает mp3.

### 5.4 Движок интервальных повторений

Подробнее в `06_SPACED_REPETITION.md`. Здесь — функциональное требование:

#### FR-SR-01
- Для каждой пары `(student_id, word_id)` хранится запись: `easiness_factor`, `interval`, `repetitions`, `next_review_at`, `last_quality`.
- При оценке карточки обновляется по упрощённому SM-2.

## 6. Нефункциональные требования

| № | Категория | Требование |
|---|-----------|------------|
| NFR-01 | Производительность | Webhook Telegram обрабатывается за <300 мс (тяжёлое — в очередь). |
| NFR-02 | Производительность | TWA: FCP < 1.5 с на 4G, LCP < 2.5 с. |
| NFR-03 | Масштабируемость | Архитектура должна держать 50 групп × 25 студентов = 1250 активных юзеров без шардинга. |
| NFR-04 | Надёжность | При падении TTS fallback-сервиса Web Speech API всё равно работает. |
| NFR-05 | Безопасность | Все требования из `07_SECURITY.md` обязательны. |
| NFR-06 | Наблюдаемость | Логи бота, API и очередей в структурированном формате (JSON) в Monolog. |
| NFR-07 | Локализация | MVP — интерфейс на русском; код готов к i18n (все строки через `__()`). |
| NFR-08 | Резервное копирование | Ежедневный бэкап БД; ретеншн 14 дней. |
| NFR-09 | Совместимость | TWA работает в Telegram iOS, Android, Desktop (последние 2 версии). |

## 7. Ограничения и решения

| Ограничение | Решение |
|-------------|---------|
| Telegram bots могут слать в одну группу не более ~20 msg/min | Rate-limit на уровне воркера, очередь отправки. |
| Web Speech API звучит по-разному на iOS/Android | Явно предупреждаем в UI; даём кнопку «переключить на серверный TTS». |
| TWA работает внутри Telegram WebView — нет полноценного PWA | Не используем Service Workers; состояние только в памяти + backend. |
| Бот может быть добавлен в публичную группу случайно | Group Lock по whitelist + уведомление админу о попытке. |

## 8. Метрики успеха (Success Metrics)

- **Engagement:** DAU/WAU ≥ 0.4 для активных групп через 2 недели после старта.
- **Retention:** ≥ 60% студентов возвращаются для повторения после push-напоминания.
- **Learning:** Средняя easiness_factor группы растёт ≥ 0.1 за месяц использования.
- **Satisfaction:** NPS учителей ≥ 40 после 1 месяца.

## 9. Схема JSON для импорта (канон)

```json
{
  "stage": 4,
  "lesson": 8,
  "vocabulary": [
    {
      "word": "Resilient",
      "translation": "Устойчивый, жизнерадостный",
      "example": "She is a resilient girl who never gives up.",
      "part_of_speech": "adjective",
      "transcription": "rɪˈzɪl.i.ənt"
    }
  ]
}
```

Валидация:
- `stage`: integer, ≥ 1, ≤ 20.
- `lesson`: integer, ≥ 1, ≤ 50.
- `vocabulary`: array, 1..200 элементов.
- `word`: string, 1..100 симв., латиница + пробелы + дефис.
- `translation`: string, 1..500 симв.
- `example`: string, 1..500 симв.
- `part_of_speech`, `transcription`: опциональны.

## 10. Out of Scope (MVP)

Следующее **не** входит в MVP и планируется на v2:
- Аудирование целых предложений с заданиями.
- Грамматические упражнения.
- Встроенный чат с учителем (учитель отвечает в Telegram — и так работает).
- Монетизация / подписки.
- Мобильное приложение вне Telegram.
- Поддержка не-английских языков.
- AI-генерация упражнений.
